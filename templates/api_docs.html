BankU API Documentation

Comprehensive REST API for BankU - Modern Banking Platform

Version 1.0 REST API JSON

Quick Start

Base URL
https://your-domain.com/api/v1
Authentication

All API endpoints require authentication. Include your session cookie or use API tokens.

Response Format

All responses are in JSON format with the following accessible status codes:

  • 200 - Success
  • 201 - Created
  • 400 - Bad Request
  • 401 - Unauthorized
  • 403 - Forbidden
  • 404 - Not Found
  • 500 - Internal Server Error

API Endpoints

Users API

GET
/api/v1/users/
Get list of all users
GET
/api/v1/users/{user_id}
Get user by ID
// Example response [ { "id": 1, "username": "john_doe", "email": "john@example.com", "created_at": "2025-01-01T00:00:00", "is_active": true } ]

Items API

GET
/api/v1/items/
Get list of available items
POST
/api/v1/items/
Create a new item
GET
/api/v1/items/{item_id}
Get item by ID
Create Item Request Body:
{ "title": "My Product", "description": "Product description", "category": "product", "profile_id": 1 }

Error Handling

All errors are returned in JSON format with the following structure:

{ "error": "Error message", "code": 400 }

Common error scenarios:

  • 400 Bad Request: Invalid request data or missing required fields
  • 401 Unauthorized: Authentication required or invalid credentials
  • 403 Forbidden: Insufficient permissions for the requested resource
  • 404 Not Found: Resource not found
  • 500 Internal Server Error: Server-side error
